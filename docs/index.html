📊 Инструкция по торговле NEW v2.0

1️⃣ Обзор таблицы торгов

Таблица отображает все закрытые сделки по всем активным стратегиям. Данные обновляются автоматически каждые 5 секунд по умолчанию.

2️⃣ Структура колонок

  • 🕐 Время — Время закрытия сделки (UTC)
  • 📝 Название — Название стратегии (Grid, Scalping, DCA и т.д.) NEW
  • 💰 Сумма — Вложено средств в позицию
  • 📈 Цена вход/выход — Цена входа и выхода
  • 💵 Профит USD — Прибыль/убыток в долларах
  • 📊 Профит % — Прибыль/убыток в процентах
  • ⏱️ Длительность — Время удержания позиции

3️⃣ Автообновление

Интервалы обновления:

  • 1 сек — Максимальное, для активного мониторинга
  • 5 сек — Оптимально (по умолчанию) ✓
  • 10 сек — Экономит трафик
  • 30 сек — Для фонового мониторинга
  • Отключено — Ручное обновление по F5

4️⃣ Пагинация

Выберите размер страницы:

  • 10 записей — Для медленного интернета
  • 20 записей — По умолчанию ✓
  • 50 записей — Для быстрого интернета
  • 100 записей — Просмотр больших объёмов
  • Все — Все доступные (требует много RAM)

5️⃣ Фильтры и экспорт

  • По стратегии — Фильтруйте по конкретной стратегии
  • По типу — Grid, Scalping, DCA и т.д.
  • По дате — Выберите диапазон дат
  • По режиму — Demo или Live торговля
  • Экспорт в Excel — Скачайте все данные в .xlsx

💡 Полезные советы

  • Установите интервал 5-10 сек для оптимального баланса
  • Используйте размер 20-50 записей на мобильных устройствах
  • Экспортируйте сделки в конце дня для анализа
  • Фильтруйте по стратегиям для сравнения производительности

⚙️ Руководство админ-панели NEW v2.0

1️⃣ Мониторинг статуса воркера

В админ-панели добавлен новый раздел для отслеживания работы торгового бота:

  • ✓ Статус — Зелёная галочка означает, что воркер работает
  • ⏱️ Последнее обновление — Времени прошло с последнего сигнала (сек)
  • 🔄 Циклов — Количество завершённых торговых циклов
  • 📦 Ордеров — Всего обработано ордеров
  • ⚠️ Ошибки — Последняя ошибка (если была)

2️⃣ Управление стратегиями

В админ-панели можно:

  • Создавать новые стратегии (Grid, Scalping, DCA, Trend, Arbitrage)
  • Редактировать параметры существующих стратегий
  • Запускать/останавливать торговлю по каждой стратегии
  • Переключаться между Demo и Live режимом
  • Устанавливать Stop-loss и risk management

3️⃣ Ключевые параметры стратегии

  • Название — Произвольное имя стратегии
  • Пара — BTC/USDT, ETH/USDT и т.д.
  • Режим — Demo (тестовая) или Live (реальная)
  • Начальный баланс — Сумма для начала (USDT)
  • Количество уровней — Для Grid: число сеток
  • Верхняя цена — Максимальная цена диапазона
  • Нижняя цена — Минимальная цена диапазона
  • Stop Loss % — Процент убытка для закрытия

4️⃣ Состояния стратегии

  • 🟢 Активна — Торговля идёт (действие: остановить)
  • 🔴 Остановлена — Ждёт команды (действие: запустить)
  • 📦 Архив — История, не торгует (действие: восстановить)
  • ⚠️ Ошибка — Произошла ошибка (действие: проверить логи)

🔒 Безопасность администратора

  • Доступ в админ-панель только для администраторов
  • Используйте надёжный пароль (минимум 15 символов)
  • Двухфакторная аутентификация рекомендуется
  • Все действия логируются и могут быть проверены

🚀 Руководство по обновлению NEW v2.0

1️⃣ Быстрый старт (2-3 минуты)

Способ 1: Ручное обновление (рекомендуется)

ssh user@server.com
cd /app
./update.sh

Что делает скрипт:

  • Создаёт резервную копию БД
  • Получает свежий код из GitHub
  • Применяет новые миграции
  • Перезагружает веб и воркер сервисы
  • Проверяет здоровье приложения

2️⃣ Способ 2: Автоматическое обновление (CI/CD)

Как работает: Каждый раз когда вы делаете git push в main ветку, GitHub Actions автоматически обновляет приложение на сервере.

Одноразовая настройка (15 минут):

  1. На сервере: сгенерировать SSH ключ 
    ssh-keygen -t ed25519 -f ~/.ssh/deploy_key -N ""
cat ~/.ssh/deploy_key  # Скопировать
  2. На GitHub: Settings → Secrets → добавить 4 переменные:
    • SERVER_HOST — Ваш IP адрес (91.210.191.80)
    • SERVER_USER — Пользователь на сервере
    • SERVER_PORT — SSH порт (обычно 22)
    • SERVER_SSH_KEY — Приватный ключ
  3. Готово! Теперь каждый git push origin main обновляет сервер

3️⃣ Откат изменений

Если что-то сломалось:

# Посмотреть доступные бэкапы
ls -lh backups/

# Восстановить конкретный бэкап
gunzip < backups/cripto_20260721_120000.sql.gz | psql -U cripto -d cripto_prod

# Или вернуться на предыдущий коммит
git log --oneline
git reset --hard <commit-hash>
./update.sh

⚠️ Устранение проблем

  • Скрипт не найден: chmod +x update.sh
  • Ошибка миграции БД: Проверить логи и запустить ./update.sh ещё раз
  • Сервис не запустился: systemctl status cripto-web
  • GitHub Actions не работает: Проверить secrets и SSH доступ
