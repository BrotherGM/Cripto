🚀 Руководство по обновлению

Для приложений на сервере с автоматическим обновлением
Cripto Grid Trading Bot v2.0 | 21.07.2026

📑 Содержание

  1. Быстрый старт (2-3 минуты)
  2. Способ 1: Ручное обновление (рекомендуется)
  3. Способ 2: Автоматическое обновление (CI/CD)
  4. Откат изменений
  5. Устранение проблем

1️⃣ Быстрый старт (2-3 минуты)

Самый простой способ обновить приложение на сервере:

✅ Минимум действий

ssh user@server.com
cd /app
./update.sh

Что делает скрипт

  • ✓ Создаёт резервную копию базы данных
  • ✓ Получает свежий код из GitHub
  • ✓ Применяет новые миграции БД
  • ✓ Перезагружает сервисы (веб и воркер)
  • ✓ Проверяет здоровье приложения
  • ✓ Показывает логи выполнения

Итоговое время: 2-3 минуты в зависимости от размера кода

2️⃣ Способ 1: Ручное обновление

✅ Рекомендуется для полного контроля

Полный процесс обновления

1Подключиться к серверу

ssh user@ваш-ip-адрес

Замените "user" на вашего пользователя, "ваш-ip-адрес" на IP или домен сервера

2Перейти в папку приложения

cd /app

3Запустить скрипт обновления

./update.sh

4Дождаться завершения

Скрипт выведет сообщение об успешном завершении:

✅ Deployment completed successfully!

5Проверить результат

curl https://your-domain.com/admin/ # Проверить доступность systemctl status cripto-web # Проверить статус сервиса

💡 Полезные команды после обновления

# Проверить статус сервисов systemctl status cripto-web cripto-worker # Посмотреть логи (последние 50 строк) journalctl -u cripto-web -n 50 # Проверить количество ордеров в БД psql -U cripto -d cripto_prod -c "SELECT COUNT(*) FROM grid_gridstrategy;"

3️⃣ Способ 2: Автоматическое обновление (CI/CD)

🤖 GitHub Actions с автоматическим развёртыванием

Каждый раз когда вы делаете git push в main ветку, GitHub Actions автоматически обновляет приложение на сервере.

Одноразовая настройка (15 минут)

1На сервере: сгенерировать SSH ключ

Подключитесь на сервер и выполните:

ssh-keygen -t ed25519 -f ~/.ssh/deploy_key -N "" cat ~/.ssh/deploy_key

Скопируйте весь вывод (это ваш приватный ключ)

2На GitHub: добавить secrets

  1. Перейти в Settings репозитория
  2. Выбрать: Secrets and variables → Actions
  3. Нажать "New repository secret"
  4. Добавить 4 переменные (см. таблицу ниже)
Имя Значение Пример
SERVER_HOST IP адрес сервера 91.210.191.80
SERVER_USER Пользователь на сервере deploy
SERVER_PORT SSH порт 22
SERVER_SSH_KEY Приватный ключ (полный текст) -----BEGIN OPENSSH...

3Готово! Теперь просто git push

git push origin main

Приложение обновится автоматически! Статус можно увидеть в GitHub → Actions

📊 Мониторинг автоматического обновления

После каждого git push перейдите на GitHub → Actions tab для просмотра статуса развёртывания.

  • 🟢 Зелёная галочка = успешно обновлено
  • 🔴 Красный крест = ошибка при обновлении
  • ⏳ Жёлтый круг = идёт обновление

4️⃣ Откат изменений

Если что-то пошло не так и нужно вернуться на предыдущую версию:

Способ 1: Восстановить БД из резервной копии

# Посмотреть доступные бэкапы
ls -lh backups/

# Восстановить конкретный бэкап (замените имя файла)
gunzip < backups/cripto_20260721_120000.sql.gz | psql -U cripto -d cripto_prod

Этот способ восстанавливает состояние БД на момент последнего автоматического бэкапа.

Способ 2: Вернуться на предыдущий коммит

# Посмотреть историю коммитов
git log --oneline

# Вернуться на нужный коммит (замените хеш)
git reset --hard a3c5f2d

# Запустить обновление
./update.sh

Способ 3: Просто перезагрузить сервис

Если приложение просто зависло:

sudo systemctl restart cripto-web
sudo systemctl restart cripto-worker

5️⃣ Устранение проблем

❌ Скрипт не найден: "command not found: update.sh"

Решение: Сделайте скрипт исполняемым

chmod +x update.sh
./update.sh

❌ Ошибка миграции БД: "relation does not exist"

Решение: Проверить логи и повторить обновление

# Посмотреть логи
journalctl -u cripto-web -n 100

# Повторить обновление
./update.sh

❌ Сервис не запустился: "Service failed"

Решение: Проверить статус и ошибки

# Проверить статус
sudo systemctl status cripto-web

# Посмотреть подробные логи
sudo journalctl -u cripto-web -n 50 --no-pager

❌ GitHub Actions не работает

Проверьте:

  • Secrets установлены правильно (Settings → Secrets)
  • SSH ключ имеет правильный формат
  • Пользователь на сервере имеет доступ к /app
  • Файл .github/workflows/deploy.yml существует в main ветке

⚠️ Если ничего не помогает

  1. Отменить последний коммит: git reset --hard HEAD~1
  2. Перезагрузить сервер: sudo reboot
  3. Проверить интернет соединение на сервере
  4. Связаться со специалистом для диагностики
